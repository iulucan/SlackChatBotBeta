# Team Charter – GreenLeaf Lausanne AI Assistant

## Mission
Build a reliable internal AI assistant that answers employee questions based on the handbook, provides accurate and trustworthy responses, includes sources when asked, and avoids hallucinations and protects sensitive data.

## Values
- Quality over quantity: We prefer a reliable, accurate bot over fancy features.
- Collaboration: We help each other learn Python and LangChain.
- Transparency: We openly share problems and progress every day,
- User focus: Every decision must serve real HR needs.
- Continuous improvement: We learn from mistakes quickly and adjust.

## Ground Rules
- We communicate daily via Slack.
- We raise issues early.
- We use branches + pull requests for all code.
- No direct commits to main.
- Decisions by consensus — otherwise PO decides.

## Communication Plan
- Slack: Daily async standups, quick questions.
- GitHub Issues: Task tracking, bug reports.
- GitHub Wiki: Documentation, decisions, ADRs.
- GitHub Projects: Kanban board — sprint tracking.
- Tuesday on-site: Sprint Review with Beat Müller.

## Definition of Done
A User Story is Done when ALL of the following are true:
- The feature works as expected.
- It has been tested by another team member.
- The answer is accurate and based on the handbook.
- Source/reference is included when needed.
- Code is pushed to GitHub via PR.
- PR is reviewed and approved.
- Task is moved to Done in the Kanban board.
